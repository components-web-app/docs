---
title: Publishable
description: Add a draft/publish workflow to any component so admins can edit privately before going live.
---

<!--
COVER:
- When to use: any component whose content an admin edits in the CMS
  (If a component's data never changes via the admin, you may not need this)

- The annotation + trait (minimal setup):
    use Silverback\ApiComponentsBundle\Annotation as Silverback;
    use Silverback\ApiComponentsBundle\Entity\Utility\PublishableTrait;

    #[Silverback\Publishable]
    class Title extends AbstractComponent {
        use PublishableTrait;
    }

- What PublishableTrait adds to your entity:
    - publishedAt: ?DateTimeInterface — null = unpublished draft
    - publishedResource: ?self — points to the published "twin"
    - draftResource: ?self — reverse; published points back to draft
    - isPublished(): bool

- API behaviour:
    - Anonymous / non-admin requests: receives the published version (or 404 if never published)
    - Admin in edit mode: receives the draft version
    - The module handles passing the correct ?published=true/false query param automatically

- Publishing: PATCH to the resource endpoint with { "publishedAt": "now" } (or whatever the field name is)
  — the bundle handles creating/updating the twin automatically

- Validation groups:
    - Default: constraints run on every save (draft and published)
    - #[Assert\NotBlank(groups: ['Title:published'])] — only runs on publish
    - The validation group name convention: ClassName:published
    - Custom group name: #[Silverback\Publishable(validationGroups: ['my_group'])]

- Security — who can publish:
    silverback_api_components:
        publishable:
            permission: "is_granted('ROLE_ADMIN')"  # default

- Customising property names (when defaults clash):
    #[Silverback\Publishable(fieldName: 'publicationDate', associationName: 'original', reverseAssociationName: 'draft')]

- Migration note: PublishableTrait adds 3 columns + 2 foreign keys; always review the generated migration

- Interaction with Mercure: when published, an update is broadcast so live visitors see new content
-->
