<!--
COVER:
- What it is: a built-in component type that acts as a proxy to another resource's collection endpoint
- Use case: "show the 3 latest blog articles on the home page" or "list all products in a category"

- Enabling/disabling (default: enabled):
    silverback_api_components:
        enabled_components:
            collection: true

- Creating a Collection resource (POST /component/collections):
    {
        "resourceIri": "/page_data/blog_articles",
        "perPage": 3,
        "defaultQueryParameters": {
            "order[createdAt]": "desc"
        }
    }

- Field explanations:
    - resourceIri: the collection endpoint to proxy
    - perPage: max items to load (works independently of client_items_per_page in API Platform)
    - defaultQueryParameters: key/value pairs appended to the proxied request
      (ordering, filtering, etc. — same params you'd pass directly to that endpoint)

- The response structure: the collection field contains a full hydra:Collection
    - hydra:member array
    - hydra:totalItems
    - hydra:view with pagination links (first/last/next/prev)

- Important: perPage vs client_items_per_page:
    - If your front-end needs to reload with different page sizes, enable client_items_per_page in API Platform
    - Use defaultQueryParameters for the default page size in that case
    - perPage alone is for fixed-size, non-user-configurable lists

- On the front-end: use useCwaCollectionResource (covered in Nuxt Module section)
    - collectionItems: the hydra:member array
    - pageModel: current page binding
    - totalPages: computed from hydra:totalItems / perPage
    - navigation helpers

- Admin: creating a Collection in the CMS
    - Add the Collection component to a component group
    - The admin tab lets you choose which resource IRI and configure perPage
-->
