---
title: Uploadable
description: Add file upload support to any component using Flysystem adapters, with optional Imagine image processing.
---

<!--
COVER:
- Use cases: image components, document downloads, video posters, any media

- Flysystem setup (required):
    - Install an adapter: composer require league/flysystem-local (or S3, GCS, etc.)
    - Register as a service with the silverback tag:
        services.php:
            $services->set(LocalFilesystemAdapter::class)
                ->args(['%kernel.project_dir%/var/storage/default'])
                ->tag(FilesystemProvider::FILESYSTEM_ADAPTER_TAG, ['alias' => 'local']);
        YAML equivalent shown too
    - The alias becomes the adapter name used in #[UploadableField]

- The annotation + trait on your entity:
    use Silverback\ApiComponentsBundle\Annotation as Silverback;
    use Silverback\ApiComponentsBundle\Entity\Utility\UploadableTrait;

    #[Silverback\Uploadable]
    class Image extends AbstractComponent {
        use UploadableTrait;

        #[Silverback\UploadableField(adapter: 'local')]
        public ?File $file = null;
    }

- Uploading a file — two options:
    1. Base64: include the base64 string in the JSON body field (preferred for REST)
    2. Multipart: POST/PUT/PATCH to /component/images/{id}/upload with multipart/form-data

- The _metadata.media_objects output (explain the structure):
    - contentUrl: publicly accessible download URL
    - fileSize, mimeType, formattedFileSize
    - width, height (images only)
    - imagineFilter (if it's a processed variant)
    - Each UploadableField gets its own key in the media_objects map

- LiipImagineBundle integration (optional but powerful):
    - Purpose: generate multiple image variants (thumbnail, hero, square, etc.) automatically
    - Data loader configuration:
        liip_imagine:
            data_loader: silverback.api_components.liip_imagine.binary.loader
    - Cache resolver using FlysystemCacheResolver:
        app.imagine.cache.resolver.local:
            class: Silverback\ApiComponentsBundle\Imagine\FlysystemCacheResolver
            arguments: [$filesystem: "@api_components.filesystem.local", $rootUrl: 'https://cdn.example.com']
    - Wiring into liip_imagine: cache: custom_cache_resolver
    - The auto-created service: api_components.filesystem.{alias}

    - Static filters on the annotation:
        #[Silverback\UploadableField(adapter: 'local', imagineFilters: ['thumbnail', 'square'])]

    - Dynamic filters via ImagineFiltersInterface:
        class Image implements ImagineFiltersInterface {
            public function getImagineFilters(string $property, ?Request $request): array {
                return ['thumbnail', 'hero'];  // request is null when called at upload time
            }
        }
    - Cache population: happens at upload time; first request generates if missing; background resolution option

- Deleting a file: send null for the field

- Migration: UploadableTrait adds a media_object table mapping for each field
-->
