---
title: Timestamped
description: Automatically record created and modified timestamps on any component.
---

<!--
COVER:
- When to use: any component where you want to know when it was created or last updated
  (often combined with Publishable for blog articles, news items, etc.)

- Minimal setup:
    use Silverback\ApiComponentsBundle\Annotation as Silverback;
    use Silverback\ApiComponentsBundle\Entity\Utility\TimestampedTrait;

    #[Silverback\Timestamped]
    class Article extends AbstractComponent {
        use TimestampedTrait;
    }

- What TimestampedTrait adds:
    - createdAt: DateTimeImmutable — set once on persist, never updated
    - modifiedAt: DateTime — updated on every flush

- These are serialized and returned in API responses automatically

- Customising field names:
    #[Silverback\Timestamped(createdAtField: 'publishedOn', modifiedAtField: 'lastEdited')]
    — if you provide custom names, you must also add getters/setters yourself (shown in full example)

- Using timestamps on the front-end:
    - Access via resource?.data?.createdAt and resource?.data?.modifiedAt
    - Format with Intl.DateTimeFormat or a date library in Vue

- Migration note: adds two timestamp columns; always immutable for createdAt
-->
