The API Components Bundle registers several Symfony console commands under the `cwa:` namespace. Run them via the standard Symfony console:

```bash
php bin/console cwa:<command>
```

In a Docker Compose setup, prefix with the service name:

```bash
docker compose exec api php bin/console cwa:<command>
```

## Available Commands

### `cwa:user:create`

Create a new user account without going through the registration API:

```bash
php bin/console cwa:user:create
```

The command prompts for email, username, and password interactively. Useful for creating the first admin account on a fresh installation before the front-end is set up.

Options:

| Option | Description |
|--------|-------------|
| `--admin` | Grant `ROLE_ADMIN` to the new user |
| `--super-admin` | Grant `ROLE_SUPER_ADMIN` to the new user |
| `--no-interaction` | Skip interactive prompts (requires all options to be passed as arguments) |

### `cwa:user:promote`

Add a role to an existing user:

```bash
php bin/console cwa:user:promote alice@example.com ROLE_ADMIN
```

### `cwa:user:demote`

Remove a role from an existing user:

```bash
php bin/console cwa:user:demote alice@example.com ROLE_ADMIN
```

### `cwa:route:generate`

Generate a `Route` record for an existing page or page data resource that doesn't have one yet:

```bash
php bin/console cwa:route:generate
```

The command is interactive — it lists resources without routes and lets you choose which to generate for. Routes are created using the same slug logic as the fixture builder.

### `cwa:mercure:publish`

Publish a Mercure update for one or more resources. Useful for triggering live updates in browsers without modifying data:

```bash
php bin/console cwa:mercure:publish /component/titles/018e-...
```

Primarily a debugging tool to confirm your Mercure setup is working.

## Running in Docker

All commands work inside the Docker `api` container. For example, to create an admin user on a fresh deployment:

```bash
docker compose exec api php bin/console cwa:user:create --admin
```
