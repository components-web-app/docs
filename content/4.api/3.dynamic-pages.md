<!--
COVER:
- The problem: you want /blog/my-first-post and /blog/second-post to use the same page template
  but render different data. A static page per article doesn't scale.

- Static vs dynamic pages recap:
    - Static: route → unique page. One page, one URL.
    - Dynamic (isTemplate: true): route → page data record → shared page template.

- AbstractPageData: your data entity class
    namespace App\Entity;
    use Silverback\ApiComponentsBundle\Entity\Core\AbstractPageData;

    #[ORM\Entity]
    #[ApiResource]
    class BlogArticle extends AbstractPageData {
        #[ORM\Column]
        public string $headline = '';

        #[ORM\Column(type: 'text')]
        public string $body = '';
    }

- The /page_data/ prefix convention:
    - Endpoints: GET /page_data/blog_articles, POST /page_data/blog_articles, etc.
    - Custom prefix: set routePrefix on #[ApiResource] (goes after /page_data/)

- What AbstractPageData provides:
    - title: ?string — overrides the page template's title for SEO
    - metaDescription: ?string — per-record meta description
    - Association back to the Page template
    - The route resolution: a Route points to a page data record which points to the template

- Setting up in the admin:
    - Create a Page with isTemplate: true, assign your layout and uiComponent
    - Create a Route pointing to the page data record (not the page directly)
    - The admin /_cwa/data section lists all page data types and their records

- nuxt.config pageData configuration:
    cwa: {
        pageData: {
            BlogArticle: {
                name: 'Blog Articles',
                metaFields: [
                    { field: 'headline', type: 'text', label: 'Headline' },
                    { field: 'publishedOn', type: 'date', label: 'Published Date' }
                ]
            }
        }
    }
    — metaFields appear as editable fields in the /_cwa/data/[type]/[iri] admin page

- Accessing page data in your page template component:
    - $cwa.resources.pageData.value — the current page data resource
    - $cwa.resources.pageDataIri.value
    - Pass page data IRI to components via props or composables

- Component positions and page data:
    - pageDataProperty on ComponentPosition: links a position to a specific field on the page data
    - Useful for: a HeroImage component whose image comes from article.heroImage

- Combining with Publishable:
    - Add #[Silverback\Publishable] to your PageData class
    - Draft articles are only visible to admins; published to all
-->
