<!--
COVER:
- What it is: a built-in component type that serialises a Symfony FormType into JSON so the front-end can render it
- Use cases: contact forms, newsletter signup, user-submitted content

- Enabling/disabling (default: enabled):
    silverback_api_components:
        enabled_components:
            form: true

- Step 1: Create your Symfony FormType as normal (no special requirements)
    class ContactType extends AbstractType {
        public function buildForm(FormBuilderInterface $builder, array $options): void {
            $builder->add('name')->add('email')->add('message', TextareaType::class);
        }
    }

- Step 2: Create a Form resource (POST /component/forms):
    { "formType": "App\\Form\\ContactType" }

- What the response contains (formView):
    - vars: form-level metadata (valid, submitted, errors, attr, etc.)
    - children: array of field views, each with:
        - vars.name, vars.full_name, vars.id
        - vars.value: current value
        - vars.errors: array of error strings
        - vars.required, vars.disabled
        - vars.block_prefixes: useful for rendering type-specific UI

- Submitting the form — two modes:
    PATCH to /component/forms/{id}/submit (field validation):
        - Send a partial object with just the fields being validated
        - Response: full formView with errors per field
        - HTTP 200 = valid, 400 = invalid
    POST to /component/forms/{id}/submit (full submission):
        - Send all fields
        - HTTP 201 = success, 400 = validation failed
        - On success: FormSuccessEvent is fired

- Handling success on the back-end (FormSuccessEvent):
    - getForm() — the Form resource
    - getFormData() — the submitted data object
    - Setting $event->result — serialized and returned to caller
    - EntityPersistFormListener: reusable listener to persist submitted data:
        class ContactListener extends EntityPersistFormListener {
            public function __construct() {
                parent::__construct(ContactType::class, ContactFormData::class, true);
            }
        }

- On the front-end: use $cwa.forms.getForm(iri) and $cwa.forms.getFormViewErrors(formIri, field)
  (covered in Nuxt Module → Forms)

- Security considerations:
    - Form submission endpoints are available to anonymous users by default (access_control in security.yaml)
    - Protect specific form types with Symfony voters if needed
-->
