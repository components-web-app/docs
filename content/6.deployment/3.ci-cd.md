A complete CI/CD pipeline for CWA has three stages: test, build, and deploy. This page shows a GitHub Actions implementation, but the concepts translate to any CI system.

## Pipeline Overview

```
push / PR
    └─ test.yml          PHP + Nuxt tests

merge to main
    └─ build.yml         Docker image builds → registry
         └─ deploy.yml   Helm upgrade / kubectl apply
```

## Test Workflow

`.github/workflows/test.yml`:

```yaml
name: Test

on: [push, pull_request]

jobs:
    php:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v4

            - name: Install PHP dependencies
              run: composer install --no-interaction --prefer-dist
              working-directory: ./api

            - name: PHP CS Fixer
              run: vendor/bin/php-cs-fixer fix --dry-run --diff
              working-directory: ./api

            - name: PHPStan
              run: vendor/bin/phpstan analyse
              working-directory: ./api

            - name: PHPUnit
              run: vendor/bin/phpunit
              working-directory: ./api
              env:
                  DATABASE_URL: postgresql://app:secret@localhost:5432/test

    nuxt:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v4
            - uses: pnpm/action-setup@v4
              with:
                  version: 11

            - name: Install dependencies
              run: pnpm install
              working-directory: ./app

            - name: Type check
              run: pnpm typecheck
              working-directory: ./app

            - name: Lint
              run: pnpm lint
              working-directory: ./app

    e2e:
        runs-on: ubuntu-latest
        steps:
            - uses: actions/checkout@v4

            - name: Start Docker Compose
              run: docker compose up -d --wait

            - name: Run migrations
              run: docker compose exec -T php bin/console doctrine:migrations:migrate --no-interaction

            - name: Load fixtures
              run: docker compose exec -T php bin/console doctrine:fixtures:load --no-interaction

            - name: Install Playwright
              run: pnpm exec playwright install --with-deps chromium
              working-directory: ./app

            - name: Run Playwright
              run: pnpm test
              working-directory: ./app

            - name: Upload test artifacts
              if: failure()
              uses: actions/upload-artifact@v4
              with:
                  name: playwright-report
                  path: app/test-results/
```

## Build Workflow

`.github/workflows/build.yml`:

```yaml
name: Build

on:
    push:
        branches: [main]
    workflow_dispatch:

jobs:
    build:
        runs-on: ubuntu-latest
        strategy:
            matrix:
                include:
                    - name: php
                      context: ./api
                      dockerfile: ./api/Dockerfile
                    - name: nuxt
                      context: ./app
                      dockerfile: ./app/Dockerfile

        steps:
            - uses: actions/checkout@v4

            - uses: docker/setup-buildx-action@v3

            - uses: docker/login-action@v3
              with:
                  registry: ghcr.io
                  username: ${{ github.actor }}
                  password: ${{ secrets.GITHUB_TOKEN }}

            - uses: docker/build-push-action@v5
              with:
                  context: ${{ matrix.context }}
                  file: ${{ matrix.dockerfile }}
                  push: true
                  tags: |
                      ghcr.io/${{ github.repository }}-${{ matrix.name }}:${{ github.sha }}
                      ghcr.io/${{ github.repository }}-${{ matrix.name }}:latest
                  cache-from: type=gha
                  cache-to: type=gha,mode=max
```

## Deploy Workflow

`.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
    workflow_run:
        workflows: [Build]
        types: [completed]
        branches: [main]

jobs:
    deploy:
        if: ${{ github.event.workflow_run.conclusion == 'success' }}
        runs-on: ubuntu-latest
        environment: production

        steps:
            - uses: actions/checkout@v4

            - name: Set up kubectl
              uses: azure/setup-kubectl@v3

            - name: Configure kubeconfig
              run: echo "${{ secrets.KUBECONFIG }}" | base64 -d > /tmp/kubeconfig

            - name: Run migrations
              env:
                  KUBECONFIG: /tmp/kubeconfig
                  SHA: ${{ github.event.workflow_run.head_sha }}
              run: |
                  kubectl run migrations-$SHA \
                      --image=ghcr.io/${{ github.repository }}-php:$SHA \
                      --restart=Never \
                      --env=DATABASE_URL="${{ secrets.DATABASE_URL }}" \
                      -- bin/console doctrine:migrations:migrate --no-interaction
                  kubectl wait --for=condition=complete job/migrations-$SHA --timeout=120s
                  kubectl delete pod migrations-$SHA

            - name: Deploy with Helm
              env:
                  KUBECONFIG: /tmp/kubeconfig
                  SHA: ${{ github.event.workflow_run.head_sha }}
              run: |
                  helm upgrade --install cwa ./chart \
                      --set php.image.tag=$SHA \
                      --set nuxt.image.tag=$SHA \
                      --values ./chart/values.production.yaml \
                      --wait
```

## GitHub Secrets Required

| Secret | Value |
|--------|-------|
| `KUBECONFIG` | Base64-encoded kubeconfig file |
| `DATABASE_URL` | Production database connection string |
| `JWT_PASSPHRASE` | JWT private key passphrase |
| `MERCURE_JWT_SECRET` | Mercure shared secret |

`GITHUB_TOKEN` is provided automatically for pushing to GHCR.

## Build Cache

The `cache-from: type=gha` / `cache-to: type=gha,mode=max` lines use GitHub Actions cache to store Docker layer cache between builds. First build is slow; subsequent builds of unchanged layers are fast.

## Staging vs Production

Use branch-specific deploy conditions and environment-specific Helm values files:

```yaml
# Deploy to staging on every merge to main
deploy-staging:
    environment: staging
    # helm upgrade ... --values chart/values.staging.yaml

# Deploy to production on version tags only (v1.2.3)
deploy-production:
    if: startsWith(github.ref, 'refs/tags/v')
    environment: production
    # Requires manual approval in GitHub Environments settings
```

## Rollback

```bash
# Via Helm
helm rollback cwa [revision]
helm history cwa   # list previous revisions

# Via kubectl (if not using Helm)
kubectl rollout undo deployment/php
kubectl rollout undo deployment/nuxt
```

Keep the previous image tag in build outputs (the git SHA) so you can reference a specific image in a manual rollback.
