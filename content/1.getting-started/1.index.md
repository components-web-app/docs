CWA gives you a working, fully-featured web application the moment you clone the template. The data structure for any website — layouts, pages, routes, and content components — is already abstracted out. Your job as a developer is to build the individual UI components and their data models. Everything else is handled.

## What you get

**A structured API built on API Platform and Symfony.** Layouts, pages, routes, and components are first-class entities. Relationships between them are data — not code — so structure can change without a deployment.

**A Nuxt module that does the heavy lifting.** Route resolution, resource fetching, Pinia store management, SSR-safe hydration, Mercure subscriptions, and a complete admin UI are all provided. You write Vue components; the module handles the plumbing.

**An inline CMS your clients can actually use.** Enter edit mode and content becomes manageable directly on the page. No separate CMS interface to learn. No rebuild to publish a change.

**Real-time updates out of the box.** Publish a component and every open browser session sees the new content instantly — no page reload, no polling. Powered by Mercure, wired up automatically.

**Draft/publish workflow for every content type.** Add one annotation to a PHP entity and you get a full draft/publish lifecycle with role-based access control and publish-time validation.

## How it's structured

A CWA application has two parts:

**The API** — a Symfony application with the API Components Bundle installed. You extend `AbstractComponent` to create content types, annotate them with `#[Publishable]`, `#[Uploadable]`, or `#[Timestamped]`, and run migrations. The bundle provides all the routing, serialization, authentication, and Mercure publishing.

**The Nuxt app** — a Nuxt application with the CWA module. You create Vue components in `app/cwa/components/`, layouts in `app/cwa/layouts/`, and page templates in `app/cwa/pages/`. The module resolves routes, fetches resources, and renders the right component tree for the current URL.

The two communicate through a typed REST/Hydra API. The component class name in PHP matches the Vue component file name — `Title.php` renders as `CwaComponentTitle.vue`. No manual mapping.

## The template repository

The [components-web-app template](https://github.com/components-web-app/components-web-app) is the starting point for every CWA project. It ships with:

- Docker Compose stack (FrankenPHP, Nuxt, PostgreSQL, Mercure, Caddy with Souin static cache)
- JWT authentication pre-wired
- Helm chart for Kubernetes deployment
- Example components, fixtures, and CI/CD scripts

Clone it, load the fixtures, and you have a working site. The docs from here tell you how to build on top of it.
