## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (or Docker Engine + Compose)
- [GitHub CLI](https://cli.github.com/) (optional, for the one-liner below)

That's it. PHP, Node, Composer, pnpm, and all other dependencies run inside Docker — nothing to install locally.

## 1. Clone the template

The template repository is your starting point. Everything you build goes on top of it.

**With the GitHub CLI:**

```bash
gh repo create my-website \
    --template="components-web-app/components-web-app" \
    --private --clone
```

## 2. Start the stack

```bash
docker compose up --detach
```

This builds and starts three services:

| Service | What it runs |
|---------|-------------|
| `php` | FrankenPHP serving the Symfony API, with Caddy (static cache via Souin, Mercure hub, Vulcain) and Xdebug |
| `app` | Nuxt SSR application with the CWA module |
| `database` | PostgreSQL |

Once running:

| URL | What's there |
|-----|-------------|
| `https://localhost/` | Your Nuxt application |
| `https://localhost/docs` | API Platform Swagger UI |

::callout{icon="i-heroicons-light-bulb"}
Useful Docker Compose commands:
- `docker compose down` — stop everything
- `docker compose logs -f [service]` — follow logs
- `docker compose build --no-cache` — rebuild images cleanly
::

## 3. Generate JWT keys

Authentication uses JWT tokens stored in HttpOnly cookies. Generate a key pair once — you'll never need to do this again for this project.

```bash
docker compose exec php sh
```

Inside the container:

```bash
mkdir -p config/jwt
php -r "print bin2hex(random_bytes(26));"   # generate a passphrase — save it
```

Copy `.env` to `.env.local` and set the passphrase:

```bash
cp .env .env.local
# Edit .env.local and set JWT_PASSPHRASE=<your passphrase>
```

Generate the keys:

```bash
openssl genpkey -out config/jwt/private.pem -aes256 -algorithm rsa -pkeyopt rsa_keygen_bits:4096
# Enter your passphrase when prompted

openssl pkey -in config/jwt/private.pem -out config/jwt/public.pem -pubout
# Enter your passphrase again
```

Exit the container:

```bash
exit
```

## 4. Load fixtures

The template includes fixtures that scaffold a complete site structure — a primary layout, a home page with component groups, and example components. Load them to get a working starting point:

```bash
docker compose exec php bin/console doctrine:fixtures:load
```

## 5. Log in

Open `https://localhost/login` in your browser. The fixtures create a default admin user — credentials are printed when you run the fixtures command.

You should now see the example site with the admin bar at the top. Click **Edit** to enter edit mode and see the inline CMS in action.

## 6. Remove the template login notice

The template ships with a modified `/app/pages/login.vue` that includes a notice for new users. Once you've logged in, delete it to use the module's built-in login page:

```bash
rm app/pages/login.vue
```

Or customise it — see [Authentication](/nuxt-module/authentication) for how to build a fully custom login page using `useLogin()`.
