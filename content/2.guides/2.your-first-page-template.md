A page template defines the **content regions** that live inside a layout's `<slot />`. It's the skeleton of a page — where the hero goes, where the main content sits, where the sidebar is.

## The File

Create `app/app/cwa/pages/PrimaryPageTemplate.vue`:

```vue
<template>
  <div class="mx-auto max-w-7xl px-4 py-10">
    <CwaComponentGroup
      reference="primary"
      :location="props.iri"
    />
  </div>
</template>

<script setup lang="ts">
import type { IriProp } from '#cwa/composables/cwa-resource'

const props = defineProps<IriProp>()
</script>
```

### Key points

**`defineProps<IriProp>()`** — every page template receives its page's IRI as a prop. The module passes this automatically.

**`CwaComponentGroup reference="primary" :location="props.iri"`** — creates a content region called `primary` owned by this page. Admins can add any components here.

## Multiple Regions

Real page templates usually have several regions:

```vue
<template>
  <div>
    <!-- Full-width hero area -->
    <CwaComponentGroup
      reference="hero"
      :location="props.iri"
      :allowed-components="['/_api/component/hero_sections']"
    />

    <!-- Main content + optional sidebar -->
    <div class="max-w-7xl mx-auto px-4 py-10 flex gap-10">
      <div class="flex-1">
        <CwaComponentGroup
          reference="main"
          :location="props.iri"
        />
      </div>
      <aside class="w-64">
        <CwaComponentGroup
          reference="sidebar"
          :location="props.iri"
        />
      </aside>
    </div>
  </div>
</template>
```

Each `CwaComponentGroup` with a unique `reference` becomes a separate region in the CMS.

## Register in nuxt.config

```ts
cwa: {
  pages: {
    PrimaryPageTemplate: {
      name: 'Primary Page',
      classes: [
        { value: '', label: 'Default' }
      ]
    }
  }
}
```

## Create a Page Record

A page template Vue file needs a matching Page record in the database.

**Via admin**: Go to `/_cwa/pages` → **Create** → set `uiComponent` to `PrimaryPageTemplate`, then assign a layout and set the route.

**Via fixture scaffold**: See [Data Fixtures](/back-end/data-fixtures) — in short:

```php
$cwa->page('home', 'PrimaryPageTemplate', layout: 'main', route: '/');
```

## Template Pages for Dynamic Content

For pages that repeat (blog, events, products), set `isTemplate: true` when creating the page record:

```php
$cwa->page('blog-template', 'PrimaryPageTemplate', layout: 'main', isTemplate: true);
```

A template page has no URL. Routes are created via PageData records that each link to this template. See [Dynamic Pages](/core-concepts/dynamic-pages) for the full pattern.
