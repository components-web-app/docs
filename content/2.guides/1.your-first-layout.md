A layout is the persistent shell of your site — header, navigation, footer. You create it once and every page uses it.

## The File

Create `app/cwa/layouts/primary.vue`:

```vue
<template>
  <div>
    <!-- Header with a navigation component group -->
    <header>
      <nav>
        <CwaComponentGroup
          reference="navigation"
          :location="$cwa.resources.layoutIri.value"
          :allowed-components="['/_api/component/navigation_links']"
        />
      </nav>
    </header>

    <!-- Page content renders here -->
    <main>
      <slot />
    </main>

    <footer>
      <p>My Site &copy; {{ new Date().getFullYear() }}</p>
    </footer>
  </div>
</template>

<script setup lang="ts">
import { useCwa } from '#imports'
const $cwa = useCwa()
</script>
```

### What's happening here

**`<slot />`** — this is where the current page template renders. The module handles injecting page content into the slot automatically.

**`CwaComponentGroup`** — marks a region where components can be added. Here it's a navigation region.
- `reference` — a stable name for this region (used to identify it in the API)
- `location` — the IRI of the resource that owns this group. For layout groups, use `$cwa.resources.layoutIri.value`
- `allowed-components` — restricts which component types admins can add. The value is the API collection URL for that component type

## Register in nuxt.config

```ts
cwa: {
  layouts: {
    primary: {
      name: 'Primary Layout',
      classes: [
        { value: '', label: 'Default' },
        { value: 'bg-gray-50', label: 'Light Background' }
      ]
    }
  }
}
```

The `classes` array defines the `uiClassNames` options an admin can select for this layout.

## Apply uiClassNames

```vue
<div :class="$cwa.resources.layout.value?.data?.uiClassNames">
  <!-- ... -->
</div>
```

Whatever class the admin selects from the `classes` array gets applied here.

## Create the Layout in the API

The Vue file alone doesn't create a Layout record in your database. You need an actual record for the module to associate pages with.

### Option A: Admin UI (simplest)

Go to `/_cwa/layouts` → **Create** → set `uiComponent` to `CwaLayoutPrimary`. The record is created immediately.

**Naming convention:** the `uiComponent` value is derived from the filename by converting it to PascalCase and prefixing with `Cwa` + the resource type. For layouts: `primary.vue` → `CwaLayoutPrimary`, `site-header.vue` → `CwaLayoutSiteHeader`. The same pattern applies to page templates and components.

### Option B: Fixture scaffold (for seeding development data)

If you want your layout created automatically when running fixtures (useful for local development and CI), add it to your fixture scaffold. See [Data Fixtures](/api/data-fixtures) for the full guide — in short:

```php
// api/src/DataFixtures/AppScaffold.php
public function build(CwaFixtureBuilder $cwa): void
{
    $cwa->layout('main', 'CwaLayoutPrimary');
    $cwa->flush();
}
```

This creates the layout record in the database on every `doctrine:fixtures:load`.
