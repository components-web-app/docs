Every page in a CWA site is described by a chain of resources stored in the API. Understanding this chain is the single most useful thing you can do before writing any code.

## The Resource Chain

::DiagramResourceChain
::

### Route

Maps a URL path to either a **Page** (static) or a **PageData** record (dynamic). Also handles redirects.

- Path: `/about-us`
- Points to: a Page or a PageData record

### Layout

The outer shell of every page — header, footer, navigation. Every page has one.

- `uiComponent: "CwaLayoutPrimary"` → renders your `app/cwa/layouts/primary.vue`
- Has its own ComponentGroups (e.g. the navigation region)
- Admins can choose from `uiClassNames` you register in `nuxt.config`

### Page

The content structure used within a layout. It declares which page template renders inside the layout's `<slot />`.

- `uiComponent: "PrimaryPageTemplate"` → renders your `app/cwa/pages/PrimaryPageTemplate.vue`
- **Static page**: one page, one URL, fixed content regions
- **Template page** (`isTemplate: true`): reused by many PageData records (one per article/product/etc.)

### ComponentGroup

A named region within a layout or page — the slot where admins add and reorder components.

- Has a `reference` string (e.g. `"hero"`, `"nav"`, `"content"`) — this is the name you use in your page template: `<CwaComponentGroup reference="hero" :location="iri" />`
- Has a `location` IRI (the layout or page it belongs to)
- Can restrict which component types are allowed (`allowedComponents`)

### ComponentPosition

One slot inside a ComponentGroup, holding exactly one component. Positions have a `sortValue` for ordering.

- Points to: one Component
- Or: a `pageDataProperty` (used in template pages to bind to a field on the PageData record)

### Component

Your custom content: a title, a paragraph, an image, a hero section, a navigation link, etc.

- `uiComponent: "Title"` → renders your `app/cwa/components/Title/Title.vue`
- Stores your content fields (title text, image URL, etc.)
- Can be publishable (draft → published workflow)

### PageData

Used with template pages to power dynamic sections like blogs, events, and product listings. Instead of a Route pointing directly to a Page, it points to a PageData record — and that record points to the Page template to use.

- Extends `AbstractPageData`, which provides `title` and `metaDescription` for SEO
- Has a `page` field pointing to the Page template (`isTemplate: true`) that should render it
- Its own Route gives it a live URL (e.g. `/blog/my-first-post`)
- Your custom fields (headline, body, image) live here and can be bound to ComponentPositions

When a Route points to a PageData record instead of a Page directly, the chain looks like this:

The URL resolves to a `BlogArticle` PageData record. The module follows the `page` reference on that record to find `BlogTemplate` — the Page that declares the content regions. `htmlContent` on `BlogArticle` is bound to a `ComponentPosition` in the template, so it renders in the right slot. The Layout wraps everything.

## _metadata

Every resource the API returns includes a `_metadata` envelope:

```json
{
  "@id": "/component/titles/abc-123",
  "@type": "Title",
  "title": "Hello World",
  "_metadata": {
    "persisted": true,
    "publishable": { "published": true, "publishedAt": "2024-01-01T00:00:00Z" },
    "mediaObjects": {}
  }
}
```

Your Vue component reads `resource?.data?.title`. The `_metadata` is used by the module internally.
