---
title: "<CwaLink />"
description: A smart link component that uses NuxtLink for internal URLs and opens external URLs in a new tab.
---

`<CwaLink>` is a drop-in replacement for `<NuxtLink>` that handles both internal and external URLs correctly. Internal paths render as `<NuxtLink>` for client-side navigation; external URLs open in a new tab with `rel="noopener noreferrer"`.

```vue
<CwaLink href="https://example.com">Visit Example</CwaLink>
<CwaLink href="/about">About us</CwaLink>
```

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `href` | `string` | Yes | The target URL. Internal paths (`/...`) use `<NuxtLink>`; absolute URLs get `target="_blank"`. |

## Usage

Use `<CwaLink>` anywhere you have a link that might be either internal or external — for example in a navigation component where the admin sets the href:

```vue
<template>
  <nav>
    <CwaLink
      v-for="item in navItems"
      :key="item.href"
      :href="item.href"
      class="text-sm font-medium hover:text-primary-500"
    >
      {{ item.label }}
    </CwaLink>
  </nav>
</template>
```

## In Rich Text

When rendering HTML from a rich-text component, `useHtmlContent` automatically replaces any `<a>` tags in the rendered HTML with `<CwaLink>` so internal links navigate without a full page reload. See [`useHtmlContent`](/nuxt-module/advanced-components/html-content).
