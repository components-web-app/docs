COVER:
- Installation:
    pnpm add @cwa/nuxt
  - Required peer modules (all must be in nuxt.config modules array):
    @nuxt/ui, @nuxt/image, @nuxtjs/seo, @pinia/nuxt, @pinia-plugin-persistedstate/nuxt, nuxt-security

- Minimum nuxt.config.ts:
    export default defineNuxtConfig({
        modules: ['@cwa/nuxt', ...peers],
        runtimeConfig: {
            public: {
                cwa: {
                    apiUrl: 'https://api.example.com',
                    apiUrlBrowser: 'https://api.example.com',
                }
            }
        }
    })

- Full cwa: config key reference:
    cwa: {
        resources: {
            MyComponent: {
                name: 'My Component',
                description: '...',
                instantAdd: true,
                defaultData: { title: 'Default Title' }
            }
        },
        layouts: {
            Primary: {
                name: 'Primary Layout',
                classes: [
                    { value: 'light', label: 'Light' },
                    { value: 'dark', label: 'Dark' }
                ]
            }
        },
        pages: {
            Primary: { name: 'Primary Page', classes: [...] }
        },
        pageData: {
            BlogArticle: {
                name: 'Blog Articles',
                metaFields: [
                    { field: 'headline', type: 'text', label: 'Headline' }
                ]
            }
        },
        pagesDepth: 2
    }

- What the module auto-provides (you do NOT write these):
    - Pages: /login, /forgot-password, /reset-password, /verify-email, /confirm-new-email
    - Layouts: the CWA chrome (admin bar, resource loader wrapper)
    - Pinia stores: resources, auth, forms, site config, UI
    - Middleware: CWA route resolution on every navigation
    - Nuxt plugin: $cwa composable injected into every component

- The $cwa global (overview; composables cover each part in depth):
    $cwa.resources  — the resource store, access by IRI
    $cwa.auth       — auth state and methods
    $cwa.forms      — form state and submission
    $cwa.ui         — admin UI state
    $cwa.siteConfig — site-wide settings from the API

- TypeScript: IriProp type for component iri props, ResourceRef<T> for typed resource access
