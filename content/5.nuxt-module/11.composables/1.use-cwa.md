```ts
import { useCwa } from '#imports'
const $cwa = useCwa()
```

`useCwa` is the global façade. Under the hood it combines the resource store, auth, and UI Pinia stores. Use it in layouts, pages, and components for any state that isn't scoped to a single component IRI.

## `$cwa.resources`

The reactive resource store for the current page.

```ts
$cwa.resources.layoutIri.value        // IRI of the current layout
$cwa.resources.layout.value           // full layout resource object
$cwa.resources.layoutLoading.value    // boolean

$cwa.resources.pageIri.value          // IRI of the current page
$cwa.resources.page.value             // full page resource object
$cwa.resources.pageLoading.value

$cwa.resources.pageDataIri.value      // IRI of current PageData (dynamic pages)
$cwa.resources.pageData.value         // the PageData resource

$cwa.resources.getResource(iri)       // fetch any resource by IRI
```

## `$cwa.auth`

```ts
$cwa.auth.isSignedIn.value            // boolean
$cwa.auth.isAdmin.value               // boolean
$cwa.auth.user.value                  // { email, username, roles, ... }
$cwa.auth.status.value                // 'SIGNED_OUT' | 'LOADING' | 'SIGNED_IN'

await $cwa.auth.signIn({ email, password })
await $cwa.auth.signOut()
await $cwa.auth.refreshUser()
```

## `$cwa.ui`

Admin UI state — useful for conditional rendering in layouts.

```ts
$cwa.ui.editMode.value                // true when admin has clicked "Edit"
$cwa.ui.selectedResource.value        // IRI of selected component, or null
$cwa.ui.isSelectedResource(iri)       // boolean
```

## `$cwa.siteConfig`

Settings fetched from the API.

```ts
$cwa.siteConfig.siteName.value
$cwa.siteConfig.maintenanceMode.value
$cwa.siteConfig.robotsEnabled.value
```

## Using in templates

`$cwa` is also injected globally by the module as a template global, so you can use it directly in any `<template>` without importing:

```vue
<template>
  <div v-if="$cwa.auth.isSignedIn.value">...</div>
</template>
```

In `<script setup>`, use `useCwa()`.
