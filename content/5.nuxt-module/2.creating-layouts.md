<!--
COVER:
- File convention:
    app/cwa/layouts/Primary.vue → registered as CwaLayoutPrimary
    File name (without .vue) = uiComponent value in the API

- Minimal layout component:
    <template>
        <div>
            <header>...</header>
            <slot />
            <footer>...</footer>
        </div>
    </template>
    — no special props needed; the page content renders into the slot

- Accessing the current layout resource:
    const layout = $cwa.resources.layout.value
    const uiClassNames = layout?.data?.uiClassNames

- Applying uiClassNames to the layout wrapper:
    <div :class="uiClassNames">...</div>
    — CSS classes defined in nuxt.config layouts.[Name].classes

- Adding a ComponentGroup inside a layout (e.g. navigation):
    <CwaComponentGroup reference="navigation" :location="$cwa.resources.layoutIri.value" />
    — reference: unique name within this location IRI
    — location: the IRI of the owning layout or page

- Multiple groups (e.g. navigation + footer):
    <CwaComponentGroup reference="navigation" :location="layoutIri" />
    <CwaComponentGroup reference="footer" :location="layoutIri" />

- Auth-conditional UI — always use ClientOnly to avoid hydration mismatches:
    <ClientOnly>
        <span v-if="$cwa.auth.isSignedIn.value">{{ $cwa.auth.user.value?.email }}</span>
        <NuxtLink v-else to="/login">Login</NuxtLink>
    </ClientOnly>

- nuxt.config layouts registration:
    cwa: {
        layouts: {
            Primary: { name: 'Primary Layout', classes: [...] }
        }
    }

- Loading states: $cwa.resources.layoutLoading.value
-->
