Low-level composable that fetches resource data and wires into the admin manager. For display components, prefer [`useCwaComponent`](/nuxt-module/composables/component/use-cwa-component) — it wraps this composable and adds a plugin system for collections, images, and custom behaviour.

```ts
import { toRef } from 'vue'
import type { IriProp } from '#cwa/composables/cwa-resource'
import { useCwaResource } from '#imports'

const props = defineProps<IriProp>()

const { getResource, exposeMeta } = useCwaResource(toRef(props, 'iri'))
const resource = getResource()

defineExpose(exposeMeta)
```

## Signature

```ts
useCwaResource(
  iri: Ref<string>,
  options?: {
    styles?: {
      multiple: boolean
      // each style's value is a class string; a string[] is also accepted and joined
      classes: Record<string, string | string[]>
    }
    name?: string            // for ui/ variant files — must match filename without .vue
    disableManager?: boolean // skip admin manager integration
    autoClass?: boolean      // default true — auto-applies uiClassNames to the root element
  }
)
```

**First argument** is a `Ref<string>` — use `toRef(props, 'iri')` to convert the `iri` prop to a ref.

## Return values

```ts
const { getResource, exposeMeta, $cwa, getCurrentStyleName } = useCwaResource(iriRef)
```

| Return | Type | Purpose |
|--------|------|---------|
| `getResource` | `() => Ref<Resource \| null \| undefined>` | Call it once in setup to get the resource ref |
| `exposeMeta` | object | Pass directly to `defineExpose` |
| `$cwa` | CwaComposable | Access to auth, resources, UI state |
| `getCurrentStyleName` | `(data) => string \| undefined` | Read the currently selected style name |
| `uiClassNames` | `ComputedRef<string[] \| undefined>` | The active style classes — auto-applied to the root element by default |

**`const resource = getResource()`** — call `getResource()` once to get the reactive resource ref. Never call it in a computed or template — call it once in `setup()`.

**`resource.value?.data?.title`** — access your PHP entity fields on `.data`.

**`resource.value` states:**
- `undefined` — loading (show skeleton)
- `null` — not found
- object — data available

**`defineExpose(exposeMeta)`** is **required** — without it the admin manager cannot find and select this component.

## Style Variants

```ts
const { getResource, exposeMeta, getCurrentStyleName } = useCwaResource(toRef(props, 'iri'), {
  styles: {
    multiple: false,
    classes: {
      'Default': '',
      'Filled': 'bg-blue-600 text-white px-6 py-2 rounded-md',
      'Outlined': 'border border-blue-600 text-blue-600 px-6 py-2 rounded-md',
    }
  }
})
const resource = getResource()
const currentStyleName = computed(() => {
  if (!resource.value?.data) return undefined
  return getCurrentStyleName(resource.value.data)
})
```

Each style's value is a **class string** (a `string[]` is also accepted and joined). The style names appear as options in the admin manager. The selected classes are automatically applied to the component's root element — no `:class` binding needed. To apply them to an inner element instead, pass `autoClass: false` and bind `uiClassNames` manually.

### Allowing several styles at once

Set `multiple: true` to let the author pick more than one style. The admin **Style** dropdown becomes a multi-select, and the selected classes are combined onto the element together (single mode stays a normal dropdown with a "Default" option):

```ts
useCwaResource(toRef(props, 'iri'), {
  styles: {
    multiple: true, // author can tick several styles
    classes: {
      Bordered: 'border border-gray-200',
      Rounded: 'rounded',
    },
  },
})
```

**Storage shape.** The resource's flat `uiClassNames: string[]` holds **one entry per selected style** — that style's class string — for **both** single and multiple select. A single selection is a one-element array (e.g. `['border border-gray-200']`); a multiple selection is an array of N (picking both above saves `['border border-gray-200', 'rounded']`).

## UI Variant Files

For a `ui/` directory alternative template, the `name` option sets the label shown in the admin "UI" selector. It does not need to match the filename — it's purely cosmetic. Without it, the selector falls back to the raw component name.

```ts
// app/cwa/components/NavigationLink/ui/YouTube.vue
const { getResource, exposeMeta } = useCwaResource(toRef(props, 'iri'), {
  name: 'YouTube'   // optional display label for the admin UI selector
})
```

See [Alternative UI Variants](/guides/alternative-ui-variants) for the full pattern.
