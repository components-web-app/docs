`useCwaForm` manages the submission lifecycle for a [Form component](/api/components/built-ins/form-component). It reads the current field values collected by [`useCwaFormInput`](/nuxt-module/composables/component/use-cwa-form-input), submits them to the API, and tracks success and error state reactively.

```ts
const { submit, submitting, success, formErrors } = useCwaForm(iri)
```

## Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `iri` | `Ref<string \| undefined>` | IRI of the Form component resource |

## Return value

| Property | Type | Description |
|----------|------|-------------|
| `submit` | `() => Promise<void>` | Submit the form ‚Äî see below |
| `submitting` | `Ref<boolean>` | `true` while the submit request is in-flight |
| `success` | `Ref<boolean>` | `true` after a successful submission (HTTP 201) |
| `formErrors` | `ComputedRef<string[]>` | Form-level error messages from the API response; updates reactively on 422 |

## `submit()`

Calling `submit()`:

1. Reads the current field values for this form from the shared store (populated by each `useCwaFormInput` field)
2. Reads `action` and `method` from the root form vars in the API response
3. POSTs or PATCHes to the submit endpoint with all field values using Symfony `full_name` keys (e.g. `contact_form[name]`)
4. On **success (201)** ‚Äî sets `success = true` and clears `submitAttempted` (hides errors in all fields)
5. On **failure (422)** ‚Äî sets `submitAttempted = true` (triggers `displayErrors` in every `useCwaFormInput` for this form); `formErrors` updates automatically from the store

`submitting` is `true` for the duration of the request.

## Example

```vue
<script setup lang="ts">
const props = defineProps<{ iri: string }>()
const iriRef = toRef(props, 'iri')

const { resource } = useCwaResource(props)
const { submit, submitting, success, formErrors } = useCwaForm(iriRef)
</script>

<template>
  <form @submit.prevent="submit">
    <!-- fields rendered by child components using useCwaFormInput -->

    <p v-for="error in formErrors" :key="error" class="text-red-500">
      {{ error }}
    </p>

    <button type="submit" :disabled="submitting">
      {{ submitting ? 'Sending‚Ä¶' : 'Submit' }}
    </button>

    <p v-if="success">Thank you ‚Äî your message has been sent.</p>
  </form>
</template>
```

## Notes

- `formErrors` reads from the root form's `vars.errors` in the store, not a local ref ‚Äî it updates automatically when the API returns a 422 response
- Field-level errors are handled by `useCwaFormInput`, not this composable
- The `submitAttempted` flag is shared state in `$cwa.forms` keyed by IRI ‚Äî no `provide`/`inject` required; all `useCwaFormInput` instances for the same form IRI respond to it automatically
