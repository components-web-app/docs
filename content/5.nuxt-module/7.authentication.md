COVER:
- Pages provided automatically — you do NOT need to create these:
    /login                            — email + password login form
    /forgot-password                  — request a password reset email
    /reset-password/[username]/[token] — set a new password
    /verify-email/[username]/[token]  — verify email on registration
    /confirm-new-email/[username]/[newEmail]/[token] — confirm email address change

- Accessing auth state anywhere in your app:
    const cwa = useCwa()              // or use $cwa in templates
    cwa.auth.isSignedIn.value         // boolean
    cwa.auth.isAdmin.value            // boolean (ROLE_ADMIN or higher)
    cwa.auth.user.value               // the current user object (null if not signed in)
    cwa.auth.status.value             // 'SIGNED_OUT' | 'LOADING' | 'SIGNED_IN'

- Signing in programmatically (e.g. custom login form):
    await cwa.auth.signIn({ username: email, password })
    — throws on invalid credentials; handle with try/catch

- Signing out:
    await cwa.auth.signOut()

- useLogin composable (wraps the login form logic):
    import { useLogin } from '#imports'
    const { model, submit, loading, error } = useLogin()
    — model.username, model.password
    — submit() triggers the sign-in and redirects on success
    — error: the error message to display

- useForgotPassword, useResetPassword — same pattern as useLogin
  — wraps the password reset flow; handles API calls, loading, error states

- useVerifyEmail, useResendVerifyEmail
  — useVerifyEmail(username, token): call on mount; returns { success, error, loading }
  — useResendVerifyEmail(username): re-send the verification email

- Protecting pages (route middleware):
    definePageMeta({ middleware: 'auth' })
    — The module provides an 'auth' middleware that redirects unauthenticated users to /login
    — Use it on any page that requires authentication

- Admin-only areas:
    if (!cwa.auth.isAdmin.value) navigateTo('/')
    — Or use the built-in middleware: definePageMeta({ middleware: 'admin' })

- Auth-conditional UI — always in <ClientOnly> to prevent hydration mismatches:
    <ClientOnly>
        <UserMenu v-if="$cwa.auth.isSignedIn.value" />
        <NuxtLink v-else to="/login">Sign in</NuxtLink>
    </ClientOnly>

- Customising the login page:
    — Override by creating app/pages/login.vue in your Nuxt app
    — It will take precedence over the module's provided page
    — Use useLogin() inside it; the module handles the API wiring

- User registration (no built-in page):
    — There is no default /register page; you build this yourself
    — POST to /users via $fetch or useFetch with your registration fields
    — The API fires the email verification flow automatically
    — After registration, redirect to /login or handle the flow yourself
