COVER:
- File convention:
    app/cwa/components/Title/Title.vue → registered as CwaComponentTitle
    Directory name = component name, file name = component name

=== PART 1: THE DISPLAY COMPONENT ===

- The mandatory pattern — every component must do these four things:
    import { useCwaResource } from '#imports'
    import type { IriProp } from '@cwa/nuxt/runtime/composables'

    const props = defineProps<IriProp>()
    const { resource, getResource } = useCwaResource(props)
    defineExpose(getResource.exposeMeta)

    — if you forget defineExpose, the admin panel can't select the component

- Accessing your API data:
    const title = computed(() => resource.value?.data?.title)
    const publishedAt = computed(() => resource.value?.data?.publishedAt)

- Rendering:
    <template>
        <div v-if="resource">
            <h1>{{ title }}</h1>
        </div>
    </template>
    — check resource first; it may be null while loading

- disableManager: opt a component out of inline admin selection
    const { resource, getResource } = useCwaResource(props, { disableManager: true })
    — use when the component is purely presentational and has no editable fields

- UI style variants (getCurrentStyleName):
    const styleName = getCurrentStyleName(resource)
    — returns the uiClassNames string set by the admin
    — use it to apply conditional classes:
    <div :class="styleName === 'dark' ? 'bg-black text-white' : 'bg-white text-black'">

- Resource loading state: resource.value === undefined (loading) vs null (not found) vs object (loaded)

=== PART 2: ADMIN MANAGER TABS ===

- File convention:
    app/cwa/components/Title/admin/Title.vue
    — auto-discovered; no registration needed

- The admin tab is shown in the manager panel when the component is selected in edit mode

- Minimal admin tab:
    import { useCwaResourceManagerTab, useCwaResourceModel } from '#imports'

    const props = defineProps<IriProp>()
    useCwaResourceManagerTab({ name: 'Content', order: 1 })
    const { model } = useCwaResourceModel(props)
    defineExpose(getResource.exposeMeta)  // required

- useCwaResourceManagerTab options:
    name: 'Content'    // tab label
    order: 1           // tab position (lower = earlier)
    disabled: false    // computed(() => !model.value.title) to conditionally disable

- useCwaResourceModel — bidirectional field binding:
    const { model } = useCwaResourceModel(props)
    — model.value.title is reactive; setting it PATCHes the API automatically
    — use v-model="model.value.fieldName" on form inputs

- Built-in form UI components (from @cwa/nuxt/runtime/components/ui/form/):
    <CwaUiFormInput v-model="model.value.title" label="Title" />
    <CwaUiFormTextarea v-model="model.value.body" label="Body" />
    <CwaUiFormToggle v-model="model.value.enabled" label="Enabled" />
    <CwaUiFormSelect v-model="model.value.category" label="Category" :options="categoryOptions" />
    <CwaUiFormFile ... />   (see Images & Media)

- Multiple admin tabs on one component:
    Create multiple files in admin/:
    app/cwa/components/Article/admin/Content.vue (order: 1)
    app/cwa/components/Article/admin/Image.vue   (order: 2)
    app/cwa/components/Article/admin/Settings.vue (order: 3)
    — each file is one tab; register with useCwaResourceManagerTab in each

- Read-only information in an admin tab:
    — You can display computed values, stats, or instructions without v-model
    — Useful for: showing the IRI, linking to external systems, displaying timestamps

=== PART 3: UI CLASS NAMES & STYLE VARIANTS ===

- What UI class names are:
    — A set of CSS class options an admin can apply to a component from the manager panel
    — Each component type can have its own set of options
    — The chosen value is stored on the component's uiClassNames field

- Registering class options in nuxt.config:
    cwa: {
        resources: {
            Title: {
                name: 'Title',
                classes: [
                    { value: '', label: 'Default' },
                    { value: 'text-center', label: 'Centered' },
                    { value: 'text-right', label: 'Right-aligned' }
                ]
            }
        }
    }

- Applying the chosen class in your component:
    Option 1 — apply directly as a class:
        <div :class="resource?.data?.uiClassNames">...</div>

    Option 2 — use as a style variant name:
        const styleName = resource?.data?.uiClassNames  // e.g. 'dark'
        <div :class="{ 'bg-black text-white': styleName === 'dark' }">

- App/cwa/components/Title/ui/ directory (future):
    — For complex variants that need more than a CSS class change
    — Each file in ui/ is a full alternative Vue template for that variant
    — If the directory exists and uiClassNames matches a filename, that template renders instead
    — Details will be documented when the feature stabilises

- The "default" class value: an empty string '' means no class applied
  (admins see it as the "Default" option)

=== PART 4: nuxt.config resources registration ===

    cwa: {
        resources: {
            Title: {
                name: 'Title Block',         // shown in Add Component dialog
                description: 'A heading or section title',  // optional helper text
                instantAdd: false,           // true = skip config dialog, add immediately
                defaultData: {               // pre-fill fields when created
                    title: 'New Title'
                },
                classes: [...]               // UI class name options (see above)
            }
        }
    }

- If a component is not registered in nuxt.config.resources, it:
    - Still renders correctly on the front-end
    - Appears in the Add Component dialog with no name or description
    - Has no UI class name options
